6.1 From Character to Function

In traditional narrative theory, we speak of "characters," "plot points," and "settings." In Nebraska 1.0, these are replaced by functional components with specific jobs.

6.2 The Killing Mechanism (K)

If the Parameter (P) is the thesis, the Killing Mechanism (K) is the antithesis. It is the structural force designated to negate the Parameter.

Definition:

```
K = ¬P under system constraints
```

Examples:

· If P = "Trust must be destroyed to be reborn," then K = "Systems that prevent destruction of trust"
· If P = "Hauntings require an architect," then K = "Forces that obscure or deny human causality"
· If P = "The hunter is the specimen," then K = "Illusions of objective separation"

Function: Without K, the system has no pressure, and thus no movement. K creates the narrative gravity well that P must escape.

6.3 Corpse Components

Elements that exist solely to be sacrificed. Their destruction provides the "Test Data" that proves the stakes of the Parameter are real.

Properties:

1. Expendable by design: Their destruction is guaranteed
2. High emotional valence: Their loss must register somatically
3. Clear connection to P: Their death must test the Parameter directly

Example: In a story with P = "Justice requires sacrifice," a Corpse Component might be an innocent bystander whose death proves the justice system's failure. Their corpse is literal test data.

6.4 Resistors

Forces (internal or external) that defend the old order. They apply friction to the Parameter, forcing it to earn its validity.

Types:

1. Internal Resistors: Character flaws, doubts, traumas
2. External Resistors: Social norms, laws, physical barriers
3. Systemic Resistors: Genre conventions, audience expectations

Function: Resistors prevent P from achieving resolution cheaply. They ensure conversion has cost.

Example: In a hero's journey, the Resistor might be the hero's own reluctance to leave home. In a horror story, it might be the town's refusal to believe in the monster.

6.5 Reactors

Elements designed to change state under pressure. They reveal the true shape of the machine by reacting to the collision of P and K.

State Change Taxonomy:

· Phase Change: Solid → Liquid → Gas (certainty → doubt → panic)
· Chemical Reaction: Two elements combine to form new compound
· Nuclear Transformation: Element changes fundamental identity

Example: A cynical detective (Reactor) exposed to genuine supernatural evil (P/K collision) undergoes nuclear transformation into a believer.

6.6 Amplifiers

Mechanisms that echo the central conflict at scale, turning a private struggle into a systemic or mythic one.

Amplification Methods:

1. Mirroring: Secondary plot reflects primary in miniature
2. Fractal Scaling: Same pattern appears at different magnitudes
3. Harmonic Resonance: Subplot vibrates at same frequency as main plot

Example: In Moby-Dick, Ahab's personal obsession (P) is amplified through: the crew (social scale), the whale (natural scale), and the sea (cosmic scale).

6.7 Deus Components

A pre-installed "Root System Rule." An expensive, high-energy component planted early (hidden in plain sight) that allows for a "miraculous" resolution without breaking the physics of the system.

The Rule: A "Deus Ex Machina" is only a failure when it is external to the system. In Nebraska 1.0, a miracle is valid if it is a Deus Component: a rule installed in the Logic Layer (Act I) that is only activated in the Resolution (Act III).

Example Setup:

· Act I: Character receives mysterious amulet with unclear function
· Act II: Amulet appears decorative, possibly forgotten
· Act III: At moment of impossible crisis, amulet activates according to rules established in Act I

Key Distinction: Not a cheat; a delayed circuit closure.

```python
        # Check 4: Conversion must reference Axiom (strict mode)
        if axiom.statement.lower() not in component.conversion_logic.lower():
            failures.append(f"Conversion doesn't reference Axiom: '{axiom.statement}'")

        if failures:
            component.validity_state = ValidityState.REJECTED
            return False, failures

        # Pass
        component.validity_state = ValidityState.Y_AXIS_PASS
        return True, [f"Valid conversion: {component.deficit_state} → {component.resolution_state}"]
```

7.4 Y²-Axis: Second-Order Validity (Clarity Field)

7.4.1 Function: Anti-Vagueness Enforcement

Y²-Axis eliminates weasel components—elements that technically pass Y-Axis but with vague, hand-wavy, or logically weak conversions.

Failure Modes Detected:

1. Vague X: "Some problem" instead of "lack of trust in institution"
2. Vague Y: "Things get better" instead of "institution reforms with transparency"
3. Logical Leap: X doesn't actually imply Y given the Axiom
4. Weasel Words: "Maybe," "could," "somehow," "perhaps"

7.4.2 Implementation Logic

```python
class Y2AxisValidator:
    """Second-order validity - clarity and necessity check"""

    def __init__(self):
        self.vague_indicators = ["some", "various", "certain", "multiple",
                                "different", "general", "overall"]
        self.weasel_words = ["might", "could", "possibly", "perhaps",
                            "maybe", "somehow"]

    def validate(self, component, axiom, narrative_state):
        # Must have passed Y-Axis first
        if component.validity_state != ValidityState.Y_AXIS_PASS:
            return False, ["Cannot validate Y² without Y-Axis pass"]

        failures = []

        # Check 1: X must be specific, not vague
        x_lower = component.deficit_state.lower()
        for vague in self.vague_indicators:
            if vague in x_lower and len(component.deficit_state.split()) < 4:
                failures.append(f"X is too vague: '{component.deficit_state}'")
                break

        # Check 2: Conversion logic must contain causal connective
        logic = component.conversion_logic.lower()
        if "because" not in logic and "therefore" not in logic and "thus" not in logic:
            failures.append("Conversion logic lacks explicit causal connective")

        # Check 3: No weasel words
        for word in self.weasel_words:
            if word in logic:
                failures.append(f"Conversion logic contains weasel word: '{word}'")

        if failures:
            component.validity_state = ValidityState.REJECTED
            return False, failures

        # Pass
        component.validity_state = ValidityState.Y2_AXIS_PASS
        return True, ["Conversion is clear, specific, and logically necessary"]
```

7.5 Y³-Axis: Integration Validity

7.5.1 Function: Systemic Coherence Check

Y³-Axis moves from component validation to system validation. It ensures all retained components function together as a single narrative machine.

Key Questions:

1. Do components connect? (Dependency checking)
2. Do any components contradict each other?
3. Are there redundant components solving the same deficit?
4. Are there isolated components (narrative islands)?

7.5.2 Implementation Logic

```python
class Y3AxisValidator:
    """Integration validity - systemic coherence check"""

    def validate(self, component, axiom, narrative_state):
        # Must have passed Y²-Axis first
        if component.validity_state != ValidityState.Y2_AXIS_PASS:
            return False, ["Cannot validate Y³ without Y²-Axis pass"]

        failures = []
        notes = []

        # Get all valid components
        valid_components = narrative_state.get_valid_components()

        # Check 1: Component should connect to other components
        connections = 0

        # Check dependencies
        for dep_id in component.depends_on:
            dep_component = narrative_state.get_component(dep_id)
            if dep_component and dep_component.is_valid():
                connections += 1
                notes.append(f"Depends on valid component: {dep_component.name}")
            elif dep_component:
                failures.append(f"Depends on invalid component: {dep_component.name}")
            else:
                failures.append(f"Depends on non-existent component: {dep_id}")

        # Check what this enables
        for enabled_id in component.enables:
            enabled_component = narrative_state.get_component(enabled_id)
            if enabled_component:
                connections += 1
                notes.append(f"Enables component: {enabled_component.name}")

        # If no connections and other valid components exist, might be isolated
        if connections == 0 and len(valid_components) > 1:
            failures.append("Component is isolated - no connections to other valid components")

        # Check 2: No contradictions with other components
        for other in valid_components:
            if other.id == component.id:
                continue

            # Check if component's Y contradicts another's X
            if (component.resolution_state and other.deficit_state and
                "not " + component.resolution_state.lower() in other.deficit_state.lower()):
                failures.append(f"Resolution contradicts {other.name}'s deficit")

            # Check for redundancy (same X and Y)
            if (component.deficit_state.lower() == other.deficit_state.lower() and
                component.resolution_state.lower() == other.resolution_state.lower()):
                failures.append(f"Redundant with {other.name} - solves same deficit")

        if failures:
            component.validity_state = ValidityState.REJECTED
            return False, failures

        # Pass
        component.validity_state = ValidityState.Y3_AXIS_PASS
        notes.append(f"Component integrates with {connections} other component(s)")
        return True, notes
```

Circular dependencies, dead ends, and contradictions represent system failures, not component failures.

7.6 Z-Axis: Meta-Evaluation

7.6.1 Function: Axiomatic Consistency Check

Z-Axis evaluates the narrative from a meta-perspective:

1. Axiom Implication: Is each component's X logically implied by the Axiom?
2. Global Consistency: Does anything in the narrative violate the Axiom?
3. Irreversible Progression: Does the narrative state move forward without regression?

7.6.2 Implementation Logic

```python
class ZAxisValidator:
    """Meta-evaluation - axiomatic consistency"""

    def validate(self, component, axiom, narrative_state):
        # Must have passed Y³-Axis first
        if component.validity_state != ValidityState.Y3_AXIS_PASS:
            return False, ["Cannot validate Z-Axis without Y³-Axis pass"]

        failures = []
        notes = []

        # Check 1: Component should reference concepts from Axiom
        if axiom.statement:
            axiom_words = set(axiom.statement.lower().split())
            component_text = f"{component.name} {component.description} {component.deficit_state} {component.resolution_state}".lower()
            component_words = set(component_text.split())

            overlapping = axiom_words.intersection(component_words)
            if len(overlapping) == 0:
                failures.append("Component doesn't reference any concepts from the Axiom")
            else:
                notes.append(f"Shares {len(overlapping)} concept(s) with Axiom")

        # Check 2: Component should move narrative state forward irreversibly
        # Resolution should not reintroduce the deficit
        if component.deficit_state and component.resolution_state:
            if component.deficit_state.lower() in component.resolution_state.lower():
                failures.append("Resolution reintroduces or perpetuates the deficit")

        # Check 3: Component should test or prove the Axiom
        if component.conversion_logic:
            if "proves" not in component.conversion_logic.lower() and "tests" not in component.conversion_logic.lower():
                notes.append("Conversion doesn't explicitly test/prove the Axiom")

        if failures:
            component.validity_state = ValidityState.REJECTED
            return False, failures

        # Pass
        component.validity_state = ValidityState.Z_AXIS_PASS
        return True, notes
```

7.7 Inversion: Adversarial Validation

7.7.1 Function: Stress Testing through Perspective Reversal

Inversion applies adversarial thinking to validate component necessity:

1. Ablation Test: Remove component—does narrative collapse?
2. Perspective Flip: View from antagonist's perspective—does logic hold?
3. Assumption Inversion: Flip key assumptions—does story still work?

```python
class InversionValidator:
    """Adversarial validation - perspective reversal"""

    def __init__(self, narrative_state):
        self.narrative_state = narrative_state

    def validate(self, component, axiom, narrative_state):
        # Must have passed Z-Axis first
        if component.validity_state != ValidityState.Z_AXIS_PASS:
            return False, ["Cannot validate Inversion without Z-Axis pass"]

        failures = []
        notes = []

        # Test 1: Ablation test - can narrative function without this component?
        dependent_count = 0
        for other in narrative_state.components.values():
            if other.id != component.id and component.id in other.depends_on and other.is_valid():
                dependent_count += 1

        if dependent_count == 0:
            # No other valid components depend on this one
            # Check if it enables anything critical
            enabling_critical = False
            for enabled_id in component.enables:
                enabled = narrative_state.get_component(enabled_id)
                if enabled and enabled.is_valid():
                    enabling_critical = True
                    break

            if not enabling_critical:
                failures.append("Ablation test: Component appears non-essential")
            else:
                notes.append(f"Component enables {len(component.enables)} other component(s)")

        # Test 2: Perspective reversal - invert X and Y
        # Check if inversion would make more sense (indicates weak original logic)
        original_x = component.deficit_state
        original_y = component.resolution_state

        if original_x and original_y:
            problem_words = ["lack", "missing", "without", "need", "want", "desire", "fear", "danger"]
            solution_words = ["have", "gain", "achieve", "resolve", "solve", "fix", "secure", "safety"]

            x_problem_score = sum(1 for word in problem_words if word in original_x.lower())
            y_solution_score = sum(1 for word in solution_words if word in original_y.lower())

            if x_problem_score < y_solution_score:
                notes.append("Inversion test: X→Y direction might be weak")

        if failures:
            component.validity_state = ValidityState.REJECTED
            return False, failures

        # Pass
        component.validity_state = ValidityState.INVERSION_PASS
        return True, notes
```

```python
def analyze_narrative_gaps(narrative_state):
    gaps = []

    # Find unresolved deficits
    all_resolutions = set()
    all_deficits = set()

    for component in narrative_state.get_valid_components():
        if component.resolution_state:
            all_resolutions.add(component.resolution_state.lower())
        if component.deficit_state:
            all_deficits.add(component.deficit_state.lower())

    unresolved = all_deficits - all_resolutions
    for deficit in unresolved:
        # Find component with this deficit
        for component in narrative_state.get_valid_components():
            if component.deficit_state and component.deficit_state.lower() == deficit:
                gaps.append({
                    "type": "unresolved_deficit",
                    "component": component.name,
                    "deficit": deficit,
                    "description": f"'{component.name}' has deficit '{deficit}' but no component resolves it"
                })

    return gaps
```
